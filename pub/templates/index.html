io.livecode.ch is an early stage prototype for turning code repositories into interactive tutorials and books, with code snippets that can be edited and run on the web.

The backend is powered by docker.io and is language-agnostic. For this prototype, runtimes with big startup costs and memory requirements are out... but I am open to supporting as many languages as possible. The docker image is kept at namin/io.livecode.ch.

Examples

  • miniKanren tutorial
  • Twelf tutorial

Your turn!

Add a .io.livecode.ch top-level folder to your GitHub repository, then checkout the URLs http://io.livecode.ch/repl/<user>/<repo> and http://io.livecode.ch/learn/<user>/<repo>.

To refresh the live installation on the server, checkout the URL http://io.livecode.ch/repl/<user>/<repo>?refresh=1.

What should the .io.livecode.ch directory contain?

  • defaults.json (example)
    a JSON file, with a property language with a value that matches one of the languages supported by this fork of the CodeMirror JavaScript frontend
  • install (example)
    an executable script that is run from the top-level of the repository when it is first installed in the system
  • run (example)
    an executable script that is run from the top-level of the repository to execute a snippet. The script takes three absolute file paths for the main code snippet, and two extra snippets, conventionally for pre and post processing
  • _site/index.html (example)
    the interactive tutorial as an HTML template: each code snippet must be enclosed in a div tag with the class live, and have a globally unique id; the data-lib attribute is an optional space-separated list of other code snippet ids that are concatenated to form the pre snippet; the post snippet is not yet used in the frontend

I hope you'll have fun, but do keep in mind that this is not at all production-ready. Please also be gentle with the server—for now, it only has 512MB total!
